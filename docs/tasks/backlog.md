# バックログ

## 高優先度

### [デザイン] アプリ全体のデザイン決定
- **目的**: 一貫性のあるUIを実現し、ブランドイメージを確立する
- **対象ファイル**: `constants/theme.ts`, 各コンポーネント, アセット
- **受け入れ基準**:
  - [ ] カラーパレットの決定（プライマリ、セカンダリ、アクセント）
  - [ ] タイポグラフィの統一（フォントサイズ、ウェイト）
  - [ ] スペーシング・余白のルール化
  - [ ] アイコンスタイルの統一
  - [ ] コンポーネントのデザインガイドライン作成
- **備考**: 決定事項は `docs/design-system.md` に記録


### [UI] キャプチャ画面の片手操作対応
- **目的**: 読書中に片手で素早くメモを取れるようにする
- **対象ファイル**: `app/(tabs)/index.tsx`, `components/capture/*`
- **受け入れ基準**:
  - [ ] 保存ボタンを画面下部に固定
  - [ ] ページ番号入力エリアを拡大
  - [ ] 感情スタンプを片手で届く位置に配置
- **参考**: US-1.1, US-1.2

### [機能] 課金要素の導入
- **目的**: アプリの収益化を実現する
- **対象ファイル**: 新規作成多数
- **受け入れ基準**:
  - [ ] 課金モデルの決定（サブスク / 買い切り / フリーミアム）
  - [ ] 無料版・有料版の機能差の定義
  - [ ] RevenueCat または expo-in-app-purchases の導入
  - [ ] 課金画面（ペイウォール）の実装
  - [ ] 購入状態の管理
  - [ ] リストア機能の実装
- **検討事項**:
  - 課金対象機能の候補:
    - クラウド同期
    - エクスポート機能
    - 本の登録数上限解除
    - OCR機能
    - 広告非表示
  - App Store / Google Play の審査要件
  - プライバシーポリシー・利用規約の整備
- **参考**: https://docs.expo.dev/versions/latest/sdk/in-app-purchases/

## 中優先度

### [機能] メモの一括紐付けの簡易化
- **目的**: ユーザーが読書後にまとめて本を紐づけられるようにする。
- **対象ファイル**: 要確認
- **受け入れ基準**:
  - [ ] メモの検索結果や未整理一覧からの一括選択
  - [ ] メモ作成時期の範囲検索機能

### [機能] メモ検索の強化
- **目的**: 目的のメモを素早く見つけられるようにする
- **対象ファイル**: `app/(tabs)/search.tsx`, `db/repositories/noteRepository.ts`
- **受け入れ基準**:
  - [ ] 著者名での検索に対応
  - [ ] 部分一致検索に対応
  - [ ] 検索結果でマッチ箇所をハイライト
  - [ ] 検索履歴の保存（最近5件）
  - [ ] 検索結果から本の詳細画面へ遷移できる（本でフィルタされた状態）
- **参考**: US-5.1, US-5.2

### [UI] 空状態のデザイン改善
- **目的**: 初回ユーザーに次のアクションを促す
- **対象ファイル**: `app/(tabs)/unsorted.tsx`, `app/(tabs)/books.tsx`
- **受け入れ基準**:
  - [ ] イラストまたはアイコンを追加
  - [ ] 具体的なアクションボタンを配置
  - [ ] 説明テキストを改善

### [UI] スプラッシュ画面のカスタマイズ
- **目的**: アプリのブランディング向上
- **対象ファイル**: `app.json`, アセット
- **受け入れ基準**:
  - [ ] オリジナルのスプラッシュ画像
  - [ ] アプリアイコンの作成
- **参考**: US-6.1

## 低優先度

### [UI] ダークモード対応の確認・調整
- **目的**: ダークモードでの視認性を確保
- **対象ファイル**: `constants/theme.ts`, 各コンポーネント
- **受け入れ基準**:
  - [ ] 全画面でダークモード時のコントラストが適切
  - [ ] 画像プレビューの背景色が適切

### [機能] メモのエクスポート
- **目的**: メモを外部に共有できるようにする
- **対象ファイル**: 新規作成
- **受け入れ基準**:
  - [ ] 1件のメモを画像として共有
  - [ ] 本単位でPDF/テキスト出力

### [技術] テストの追加
- **目的**: リグレッションを防ぐ
- **対象ファイル**: `__tests__/`, `jest.config.js`
- **受け入れ基準**:
  - [ ] Repository のユニットテスト
  - [ ] 主要画面の統合テスト
- **参考**: docs/testing-strategy.md
