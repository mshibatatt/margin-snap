# ユーザーストーリー

## 概要

本ドキュメントでは、Margin Snap の機能をユーザーストーリー形式で定義します。
各ストーリーには受け入れ基準（Acceptance Criteria）を含みます。

---

## Epic 1: メモ作成（キャプチャ）

### US-1.1: 写真付きメモを作成する

**ストーリー**
> 読書中のユーザーとして、
> 本のページを写真に撮ってメモを作成したい。
> なぜなら、後で見返したときに文脈を思い出せるからだ。

**受け入れ基準**
- [ ] カメラを起動して写真を撮影できる
- [ ] 撮影した写真をプレビューで確認できる
- [ ] 写真を撮り直すことができる
- [ ] 写真があればページ番号なしでも保存できる
- [ ] 保存後、フォームがリセットされる

---

### US-1.2: ページ番号のみでメモを作成する

**ストーリー**
> 読書中のユーザーとして、
> ページ番号だけを素早くメモしたい。
> なぜなら、写真を撮る余裕がないときでも記録を残したいからだ。

**受け入れ基準**
- [ ] テンキーでページ番号を入力できる
- [ ] ページ番号があれば写真なしでも保存できる
- [ ] 写真なし・ページなしの状態では保存ボタンが無効になる
- [ ] 保存できない理由がユーザーに表示される

---

### US-1.3: 感情スタンプを付ける

**ストーリー**
> 読書中のユーザーとして、
> その瞬間の感情をスタンプで記録したい。
> なぜなら、後で感情を手がかりにメモを探せるからだ。

**受け入れ基準**
- [ ] 5種類の感情スタンプ（🙂 🤔 😮 ✨ 😢）が表示される
- [ ] 1つのスタンプを選択できる
- [ ] 選択済みのスタンプを再タップで解除できる
- [ ] スタンプは任意（なしでも保存可能）

---

### US-1.4: 短文メモを追加する

**ストーリー**
> 読書中のユーザーとして、
> 短いメモを追加したい。
> なぜなら、その瞬間の気づきを言葉で残したいからだ。

**受け入れ基準**
- [ ] テキスト入力フィールドがある
- [ ] 1-2行程度の入力を想定したUI
- [ ] メモは任意（なしでも保存可能）
- [ ] 入力中でもカメラ・ページ番号の操作が可能

---

## Epic 2: メモ一覧・詳細

### US-2.1: メモ一覧を表示する

**ストーリー**
> ユーザーとして、
> 作成したメモを一覧で確認したい。
> なぜなら、過去の記録を振り返りたいからだ。

**受け入れ基準**
- [ ] メモがカード形式で一覧表示される
- [ ] 各カードに以下が表示される
  - 写真サムネイル（あれば）
  - ページ番号（あれば）
  - 感情スタンプ（あれば）
  - 短文メモ（あれば、1行に省略）
  - 作成日時
- [ ] デフォルトは新しい順で表示
- [ ] メモがない場合は空状態メッセージが表示される

---

### US-2.2: メモをソートする

**ストーリー**
> ユーザーとして、
> メモの並び順を変更したい。
> なぜなら、状況に応じて見やすい順序が異なるからだ。

**受け入れ基準**
- [ ] 「新しい順」「ページ順」の切り替えができる
- [ ] ソート状態が維持される
- [ ] ページ番号がないメモはページ順では最後に表示される

---

### US-2.3: メモをフィルタする

**ストーリー**
> ユーザーとして、
> 条件を指定してメモを絞り込みたい。
> なぜなら、特定のメモを効率的に見つけたいからだ。

**受け入れ基準**
- [ ] 期間フィルタ（今日 / 過去7日 / 過去30日 / すべて）
- [ ] 感情スタンプフィルタ（複数選択可）
- [ ] メモ状態フィルタ（すべて / 未整理のみ / 本ありのみ）
- [ ] 本フィルタ（特定の本を選択）
- [ ] フィルタを組み合わせて使用できる
- [ ] フィルタをリセットできる
- [ ] フィルタ適用中は件数が表示される

---

### US-2.4: メモ詳細を表示する

**ストーリー**
> ユーザーとして、
> メモの詳細を確認したい。
> なぜなら、写真を大きく見たり、全文を確認したいからだ。

**受け入れ基準**
- [ ] メモカードタップで詳細画面に遷移
- [ ] 写真がフルサイズで表示される
- [ ] 写真をピンチで拡大できる
- [ ] すべてのメモ情報が表示される
- [ ] 戻るボタンで一覧に戻れる

---

### US-2.5: メモを編集する

**ストーリー**
> ユーザーとして、
> 作成したメモを編集したい。
> なぜなら、後から内容を修正したいことがあるからだ。

**受け入れ基準**
- [ ] 詳細画面から編集モードに切り替えられる
- [ ] ページ番号、短文メモ、感情スタンプを編集できる
- [ ] 写真の変更・追加はできない（仕様上の制限）
- [ ] 編集をキャンセルできる
- [ ] 変更を保存できる

---

### US-2.6: メモを削除する

**ストーリー**
> ユーザーとして、
> 不要なメモを削除したい。
> なぜなら、整理して見やすくしたいからだ。

**受け入れ基準**
- [ ] 詳細画面から削除できる
- [ ] 削除前に確認ダイアログが表示される
- [ ] 削除後は一覧画面に戻る
- [ ] 削除されたメモは一覧から消える

---

### US-2.7: 複数メモを一括操作する

**ストーリー**
> ユーザーとして、
> 複数のメモをまとめて操作したい。
> なぜなら、効率的に整理したいからだ。

**受け入れ基準**
- [ ] 一覧で複数選択モードに切り替えられる
- [ ] メモを複数選択できる
- [ ] 選択したメモを一括で本に割り当てられる
- [ ] 選択したメモを一括で削除できる
- [ ] 選択を解除できる

---

## Epic 3: 本管理

### US-3.1: 本を追加する

**ストーリー**
> ユーザーとして、
> 新しい本を登録したい。
> なぜなら、メモを本ごとに整理したいからだ。

**受け入れ基準**
- [x] 本一覧画面から追加ボタンをタップ
- [x] タイトルを入力できる（必須）
- [x] 著者を入力できる（任意）
- [x] 巻数を入力できる（任意、例: "1", "上", "前編"）
- [ ] 表紙画像を設定できる（任意）
- [x] 保存後、本一覧に追加される

---

### US-3.2: 本一覧を表示する

**ストーリー**
> ユーザーとして、
> 登録した本を一覧で確認したい。
> なぜなら、どの本にどれだけメモがあるか把握したいからだ。

**受け入れ基準**
- [x] 本がカード形式で一覧表示される
- [x] 各カードに以下が表示される
  - 表紙サムネイル（あれば）
  - タイトル（巻数があれば併記）
  - 著者（あれば）
  - メモ数
  - 最終更新日
- [x] 本がない場合は空状態メッセージが表示される

---

### US-3.3: 本のメモを表示する

**ストーリー**
> ユーザーとして、
> 特定の本に紐づくメモだけを見たい。
> なぜなら、その本の読書記録を振り返りたいからだ。

**受け入れ基準**
- [ ] 本カードタップで条件付きメモ一覧へ遷移
- [ ] その本のメモだけがフィルタされている
- [ ] 通常のフィルタ・ソート機能も使用可能

---

### US-3.4: 本を編集する

**ストーリー**
> ユーザーとして、
> 本の情報を編集したい。
> なぜなら、タイトルを修正したり表紙を変更したいからだ。

**受け入れ基準**
- [x] 本詳細画面から編集モードに入れる
- [x] タイトルを変更できる
- [x] 著者を変更できる
- [x] 巻数を変更できる
- [ ] 表紙画像を変更・削除できる
- [x] 編集をキャンセルできる

---

### US-3.5: メモを本に割り当てる

**ストーリー**
> ユーザーとして、
> 未整理メモを本に紐づけたい。
> なぜなら、後から整理したいからだ。

**受け入れ基準**
- [ ] メモ詳細画面から本を選択できる
- [ ] 既存の本から選択するか、新規作成できる
- [ ] 割り当て後、メモは「未整理」から外れる
- [ ] 複数メモを一括で同じ本に割り当てられる

---

## Epic 4: 未整理メモ

### US-4.1: 未整理メモを表示する

**ストーリー**
> ユーザーとして、
> 本に紐づいていないメモを確認したい。
> なぜなら、整理すべきメモを把握したいからだ。

**受け入れ基準**
- [ ] 未整理タブをタップすると条件付きメモ一覧へ遷移
- [ ] 「未整理のみ」フィルタが初期適用されている
- [ ] フィルタを解除すると全メモが表示される

---

## Epic 5: 検索

### US-5.1: メモを検索する

**ストーリー**
> ユーザーとして、
> キーワードでメモを検索したい。
> なぜなら、特定の記録を素早く見つけたいからだ。

**受け入れ基準**
- [ ] 検索バーにテキストを入力できる
- [ ] 以下の項目が検索対象となる
  - 短文メモのテキスト
  - 本タイトル
  - ページ番号（文字列として）
- [ ] リアルタイムに検索結果が更新される
- [ ] 検索結果は条件付きメモ一覧として表示される

---

### US-5.2: 検索とフィルタを組み合わせる

**ストーリー**
> ユーザーとして、
> 検索結果をさらに絞り込みたい。
> なぜなら、より正確に目的のメモを見つけたいからだ。

**受け入れ基準**
- [ ] 検索結果にフィルタを適用できる
- [ ] フィルタと検索の両方が適用された結果が表示される
- [ ] 件数が表示される

---

## Epic 6: システム

### US-6.1: アプリを起動する

**ストーリー**
> ユーザーとして、
> アプリを素早く起動したい。
> なぜなら、読書中の一瞬を逃したくないからだ。

**受け入れ基準**
- [ ] スプラッシュ画面が表示される
- [ ] 起動後はキャプチャタブが表示される（最速記録のため）
- [ ] 前回のデータが保持されている

---

### US-6.2: カメラ権限をリクエストする

**ストーリー**
> ユーザーとして、
> カメラ権限を求められたとき、理由を理解したい。
> なぜなら、なぜ権限が必要か分からないと不安だからだ。

**受け入れ基準**
- [ ] 初回撮影時に権限リクエストが表示される
- [ ] 権限の使用目的が説明される
- [ ] 権限を拒否してもアプリは使用できる（ページ番号のみ）
- [ ] 設定画面から権限を変更する導線がある

---
